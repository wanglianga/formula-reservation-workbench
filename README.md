# 母婴宝 · 奶粉预约系统

面向母婴门店的奶粉预约工作台，覆盖**店员接待**、**仓管入库**、**督导分析**三类角色完整业务流程。温和清爽的设计风格，清晰分隔宝宝信息、商品库存、赠品规则三大模块，让业务操作一目了然。

## 原始需求

> 母婴门店需要奶粉预约页面，Vue3 可以把宝宝档案、奶粉阶段、预约库存和赠品资格放进店员工作台。业务对象包括宝宝月龄、过敏备注、品牌段位、预约罐数、到店时间、会员等级、赠品活动、核销记录和缺货登记。店员接待顾客时要看到是否适合当前段位、是否已有预约、赠品是否可领；仓管更新到货数量和效期；督导关注活动核销和异常缺货。页面要温和清楚，宝宝信息、商品库存和赠品规则不能搅在一块。

## 技术栈

- **框架**: Vue 3.4 + TypeScript 5.3
- **构建**: Vite 5.2
- **状态管理**: Pinia 2.1
- **路由**: Vue Router 4
- **样式**: Tailwind CSS 3.4
- **图标**: lucide-vue-next
- **容器化**: Docker + Nginx

## 启动方式

### 前置要求

- **Node.js** ≥ 20（推荐 20.x LTS）
- **npm** ≥ 10 或 **pnpm** ≥ 8
- **Docker** ≥ 24 + **Docker Compose** ≥ 2.20（Docker 方式需要）

### 方式一：本地开发启动（推荐日常开发）

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发服务器

```bash
npm run dev
```

访问地址：**http://localhost:5173**

Vite 默认端口 5173，如被占用会自动切换到下一个可用端口，请查看终端输出。

---

### 方式二：Docker 一键启动（推荐生产/演示）

本项目支持根目录 Docker Compose 一键构建并启动。

#### 1. 构建并启动容器

```bash
docker compose up --build
```

后台运行模式：

```bash
docker compose up --build -d
```

#### 2. 访问服务

访问地址：**http://localhost:8080**

#### 3. 停止与清理

```bash
docker compose down
```

## 功能模块

### 👩‍💼 店员工作台（默认入口 /staff）

| 模块 | 功能说明 |
|------|----------|
| **宝宝档案卡** | 会员下拉选择，宝宝卡片展示姓名/月龄/性别/过敏备注/会员等级，过敏红色高亮标签 |
| **段位匹配推荐** | 自动根据月龄和过敏备注计算12款SKU的匹配度，绿色"适配"/黄色"月龄不匹配"/红色"过敏风险" |
| **预约创建表单** | 罐数步进器、库存实时校验、到店时间选择，已存在待取货时警告提示 |
| **赠品资格面板** | 根据会员等级+罐数自动计算可领赠品，展示名称/数量/获取原因 |
| **历史预约列表** | 三态徽章（待取货/已完成/已取消），待取货支持完成/取消操作 |

### 📦 仓管工作台（/warehouse）

| 模块 | 功能说明 |
|------|----------|
| **库存总览** | 12款SKU库存卡片，显示罐数+状态标签（缺货/紧张/充足）+效期进度条 |
| **到货入库登记** | 自动生成批次号，填写商品/数量/生产日期/过期日/供应商，校验过期晚于生产 |
| **效期管理** | 三Tab切换（全部/临期预警/已过期），天数随级别变色，过期批次红底醒目 |
| **批次明细** | 支持按SKU筛选，默认按过期日排序，先进先出清晰管理 |

### 📊 督导工作台（/supervisor）

| 模块 | 功能说明 |
|------|----------|
| **活动核销进度** | 活动卡片网格+环形进度图+核销率分级标签+TOP5赠品发放条形图 |
| **异常告警看板** | 库存预警/临期预警/过期批次/未处理缺货四类告警，彩色卡片一目了然 |
| **核销记录查询** | 时间范围+状态双重筛选，导出CSV按钮（模拟），三态徽章展示 |
| **缺货登记追踪** | 待处理/已解决/已取消三态，支持一键标记已解决，顶部统计汇总 |

### 🧠 核心业务规则

- **段位匹配**：0-6月→1段、6-12月→2段、12-36月→3段、36月以上→4段，过敏品牌包含则强制标红
- **赠品计算**：会员等级固定赠品 + 满3/6/12罐阶梯赠品，自动汇总去重
- **库存扣减**：创建预约时从最早批次先扣，FIFO先进先出
- **效期预警**：>90天绿/30-90天黄/<30天红/已过期红底白字加粗
- **核销联动**：预约完成自动标记赠品已核销，取消预约自动回滚库存+标记赠品过期

## 目录结构

```
src/
├── components/
│   ├── common/          # 通用组件（RoleTabs、StatCard、ProgressRing、DataTable）
│   ├── staff/           # 店员工作台子组件
│   ├── warehouse/       # 仓管工作台子组件
│   └── supervisor/      # 督导工作台子组件
├── composables/         # 业务逻辑（段位匹配/效期检查/赠品计算）
├── data/                # Mock种子数据
├── pages/               # 页面级组件（Layout + 三个工作台）
├── router/              # 路由配置
├── stores/              # Pinia 状态管理（member/inventory/appointment）
├── types/               # TypeScript 全局类型定义
├── App.vue
├── main.ts
└── style.css
```

## 注意事项

1. 本项目为**纯前端演示**，使用内置 Mock 种子数据，刷新页面后数据会重置为初始状态
2. 所有操作均为本地即时生效，不依赖后端服务，适合演示和原型验证
3. 若需要连接真实后端，可在 `src/stores/` 中替换 action 实现为真实 API 调用
4. Docker 方式使用 Nginx 托管静态产物，镜像约 80MB，启动时间 < 3秒
